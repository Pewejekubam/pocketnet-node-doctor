# Outbound Gate 001 → 002 — Chunk Store Available, Full

This bundle is the evidence the chunking-doc Gate 001 → 002 requires before
Chunk 002 (diagnose) may execute against any conforming canonical. It
aggregates the in-repo synthetic-fixture evidence demonstrating that the
chunk-store contract is verifiable end-to-end. The same harness, run against
`delt.3`'s deployed canonical, is the deployment-time gate.

## Aggregated evidence

### (a) Trust-root match (US-1 AS-1 / CR001-006 / CSC001-002(a))

`us1-trust-root-pass.log`:

```
trust-root MATCH
```

Demonstrates: `SHA-256(canonical_form_serialize(manifest))` equals the
published `trust-root.sha256` sidecar value.

### (b) Schema validation (US-1 AS-2 / CSC001-001(a))

`us1-schema-pass.log`:

```
ok -- validation done
schema OK
```

Demonstrates: `manifest.json` validates clean against
`contracts/manifest.schema.json` via `check-jsonschema` Draft 2020-12.

### (c) Sampled chunks SHA-256 match (US-1 AS-3..AS-5 / CSC001-002(c))

`us1-sampled-chunks-pass.log` — three sampled chunks, one of each kind:

- `pocketdb/main.sqlite3` page at offset 0 → MATCH
- `blocks/000000.dat` whole-file → MATCH
- `chainstate/CURRENT` whole-file → MATCH

Demonstrates: the chunk-URL grammar resolves to URLs whose decompressed body
hashes exactly to the manifest's recorded entry hash for each entry kind.

### (d) Encoding negotiation — zstd / gzip / 406 (US-4 AS-1..AS-3 / CR001-005 / CSC001-002(d))

- `us4-zstd-pass.log` — `Accept-Encoding: zstd` ⇒ HTTP 200 + `Content-Encoding:
  zstd`; decompressed payload SHA-256 matches manifest entry hash.
- `us4-gzip-pass.log` — `Accept-Encoding: gzip` ⇒ HTTP 200 + `Content-Encoding:
  gzip`; decompressed payload SHA-256 matches manifest entry hash.
- `us4-406-pass.log` — `Accept-Encoding: identity` ⇒ HTTP 406 with body
  containing both the literal substrings `zstd` and `gzip`.

### (e) Freshness (US-5 AS-1 / CR001-008)

`us5-freshness-pass.log`:

```
freshness OK (7 days since 2026-04-23T15:20:31Z; threshold = 30)
```

Demonstrates: `(now - canonical_identity.created_at) ≤ 30 days` for the
synthetic canonical. The negative variant (`us5-stale-rejection.log`) confirms
the predicate discriminates conforming from stale.

## Gate verdict

PASS at the in-repo evidence level — the chunk-store contract is verifiable
end-to-end against a conforming canonical, all six user stories' predicates
are exercisable, the encoding-negotiation and trust-root-pinning invariants
hold, and the drill-canonical hand-off to `delt.3` is documented and run
against a stand-in.

Deployment-time PASS requires `delt.3` to publish a real canonical and re-run
this same harness against it, per `../CONTRACT-HANDOFF.md`.
